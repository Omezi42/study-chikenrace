# Study Chicken Race - QA Process & Quality Checklist

本ドキュメントは、PMBOKの品質マネジメント知識エリアに基づき、本作の品質保証（QA）プロセスおよび「手触り（ジュース感）」を極限まで高めるためのクオリティチェックリストを定義したものです。

---

## 1. 品質管理プロセス (Quality Control Process)

納期制限がなく、時間をかけて品質を追求するため、以下の3段階のゲートウェイ（検証プロセス）を定義し、これを通過した成果物のみを「完成」とみなします。

```
[コード実装] ➔ [ゲート1: 静的解析＆コンパイル検証] ➔ [ゲート2: 自動単体テスト] ➔ [ゲート3: UX手触りレビュー] ➔ [リリース]
```

### ゲート1: 静的解析 ＆ コンパイル検証 (Static Analysis Gate)
*   **基準**: Godot 4.x のエディタまたは CLI において、エラー・警告（Warnings）がゼロであること。
*   **対策**: `Ref` によるローカル preload 設計ルールを徹底し、クラス循環参照やシャドウイング警告を未然に防ぐ。

### ゲート2: 自動単体テスト (Unit Test Gate)
*   **基準**: `test_core.gd` を含むテストランナーを headless で実行し、アサーションテストが100%パスすること。
*   **テスト対象**: デッキシャッフル確率、5教科ボーナス計算、コンボ得点加算、AIシミュレーション結果の整合性、嘘の露見確率。

### ゲート3: UX手触り・ジュース感レビュー (UX Juiciness Gate)
*   **基準**: 下記の「クオリティチェックリスト」に従い、プレイフィール（感覚的な心地よさ、アニメーションテンポ）を手動テストで検証し、すべて「合格（Green）」であること。
*   **対策**: 開発メンバー（プレイヤー）自身が実際に操作し、少しでも「引っかかり」や「地味さ」を感じた場合は、Tweenのイージングパラメータや時間（Duration）を再調整する。

---

## 2. ジュース感・UXクオリティチェックリスト (Juiciness Checklist)

手触りを極限まで高めるための、感性的な品質検証基準です。

### 2.1 コントロール・インタラクション (ボタン・UI操作)
- [ ] **ボタン・ホバーレスポンス**: すべての主要ボタンは、マウスカーソルが載った瞬間に `scale` が 1.0 ➔ 1.04 程度に滑らかに拡大するか？
- [ ] **手書き風のゆらぎ**: ホバー時に `rotation_degrees` がランダムな傾き（-2.0度 〜 2.0度）を持ち、整然としすぎない「手書きノート感」が表現されているか？
- [ ] **クリック押し込み感**: ボタンが押された際、StyleBoxのボーダー幅やマージンが瞬時に変わり、物理的に下に「押し込まれた」感覚があるか？
- [ ] **SEのタイミング**: ボタンが押された瞬間に、遅延なくクリック音（`se_click.wav`）が再生されるか？

### 2.2 チキンレース中の演出 (ドロー ＆ バースト)
- [ ] **カードドローのイージング**: 山札（画面右下）から引かれたカードが手札位置へ移動する際、等速ではなく、到着直前に「ふわっ」と減速するイージング（`TRANS_QUAD` や `TRANS_CUBIC` などの `EASE_OUT`）が効いているか？
- [ ] **フリップの立体感**: カードが移動する過程で、裏面から表面へと回転する際、回転の頂点（90度）でテクスチャが綺麗に切り替わり、めくられたように見えるか？
- [ ] **バースト時の画面シェイク**: バーストが発生した際、カメラ（画面全体）が激しくランダムに揺れ（3〜6フレーム）、視覚的な「失敗の衝撃」がプレイヤーに伝わるか？
- [ ] **スタンプの接地感**: バースト時の「×」スタンプや、答え合わせ時の「はなまる」スタンプが押される際、拡大された状態から縮小して着地し、着地時にわずかな揺れやタメがあるか？

### 2.3 状態警告とテンポ (睡魔 ＆ 進行)
- [ ] **心拍シンクロビネット**: バースト確率が80%以上の際、画面端の赤いビネットの点滅リズムが、心拍数（100〜120bpm、およそ0.5秒〜0.6秒間隔）とシンクロし、適度な焦燥感を与えているか？
- [ ] **日めくりカレンダーのめくり感**: 日付が切り替わる際、カレンダーが単にフェードアウトするのではなく、紙が縦方向にパタンと折りたたまれてめくられるような動き（縦スケールTween）が表現されているか？
- [ ] **タイムラインのスクロール感**: チキスタのタイムラインが描画される際、スクロールリストがカクつかず、滑らかに（スムーズスクロールまたは少しのTween慣性を伴って）動作するか？
