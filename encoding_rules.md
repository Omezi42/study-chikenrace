# 文字コード（エンコーディング）運用ガイドライン

本プロジェクトでは、スクリプトファイル内の日本語コメントや表示用の定数文字列が多数含まれます。
開発環境や評価・レビュー環境での表示文字化けを防ぐため、以下の運用ルールを定めます。

---

## 1. 基本エンコーディングルール

* **保存形式**: すべてのスクリプトファイル（`.gd` 形式）は **UTF-8（BOMなし）** で保存してください。
  * Godot Engine 4.x は UTF-8（BOMなし）を標準として想定しており、他の形式（Shift_JIS / UTF-8 with BOM など）ではコンパイルエラーや予期せぬ挙動の原因となります。
* **エンコーディング宣言ヘッダの付与**: 日本語の文字列リテラルやコメントを含む重要なスクリプトファイルの先頭には、以下のエンコーディング宣言を必ず記述してください。
  ```gdscript
  # -*- coding: utf-8 -*-
  ```
  * これにより、Windows 環境のエディタ（VS Code やメモ帳など）や静的解析ツールが、ファイルを誤って Shift_JIS/CP932 と判定してデコードするのを防止できます。

---

## 2. コマンドライン（PowerShell）での文字化け対策

Windows 環境の PowerShell は、デフォルトでファイルの読み込み時にシステム標準のロケール（Shift_JIS/CP932）を使用するため、UTF-8 ファイルを普通に `Get-Content` すると文字化けして表示されます。

これは**ファイルの中身が壊れているのではなく、表示コマンドが誤ったコードページで解釈しているだけ**です。以下の手順で正しく表示できます。

### 対処法 A: エンコーディングを明示的に指定する（推奨）
PowerShell でファイルを表示する際は、`-Encoding UTF8` オプションを追加してください。
```powershell
Get-Content -Path src\autoload\Constants.gd -Encoding UTF8
```

### 対処法 B: コードページを切り替える
コンソール自体のコードページを UTF-8（65001）に切り替えるコマンドを実行します。
```powershell
chcp 65001
```

---

## 3. 今後の文字列追加・修正フロー

表示用の文字列や称号などを新たに追加・変更する場合は、以下の3ステップを同時に実施してください。

1. `Constants.gd` に `TITLE_*` などの定数として定義を追加（日本語文字列を直接コードに書くのは Constants 内に限定）。
2. 使用先（`ScoreEvaluator.gd` 等）では必ず `Constants.TITLE_*` 参照を使用。
3. `test_core.gd` の期待値アサーションでも `Constants.TITLE_*` 参照を使用し、ヘッドレスでテストをパスさせる。
