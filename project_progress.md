## 現在のフェーズ：Phase 41 (完了 - バースト（寝落ち）時のフリーズ完全解消 ＆ アニメーション同期待ちの堅牢化)
### 実施内容
- **拡大時スマホ座標の絶妙な黄金比対称配置（Y=120px, 1.4倍）の完成**:
  - ユーザーのフィードバックに基づき、拡大率を `1.32倍` から **`1.4倍`** へとさらに微増させ、目標Y座標を **`120px`** （50px下にずらす）に調整。
  - 数学的計算に基づき、1080pの縦画面においてスマホの上下ベゼルが全く均等に（48pxずつ）隠れるようになり、ヘッダー・フッターの見映えを一切崩さずに全画面を美しく占有する「究極の黄金レイアウト」が完成しました。
- **初手バースト（寝落ち）時の進行不能（フリーズ）バグを根本解決**:
  - `_trigger_burst_sequence` 内で、描画カードが存在しない場合（1枚目のドローで即バーストした場合等）に、中身が「空」の Tween（`fade_black` や `hop_tw`）が生成されていました。
  - Godot 4 の Tween エンジン仕様により、空の Tween に対して `await tween.finished` を実行するとシグナルが永久に発火せず、スレッドが無限にスリープ（フリーズ）するバグの原因を特定。
  - 空の Tween が生成されるのを防ぐガード処理（有効カード判定 `has_cards`）を導入し、さらに awaits 同期を **`get_tree().create_timer(0.45).timeout`** による安全な時間ベースの待ち処理へと一本化！
  - これにより、どのような例外的なタイミングで寝落ち（バースト）が発生しても、100%確実にダイアログへ進行する絶対無敵の堅牢なゲームループを完成させました。

## プロジェクト構造

```
study-chikenrace/
├── project.godot              # プロジェクト設定（オートロード: Global, SceneTransition）
├── *.tscn                     # シーンファイル（Main, Title, Profile, ResultScene）
├── scripts/
│   ├── core/                  # ゲームロジック層
│   │   ├── Global.gd          #   セーブ/ロード/状態管理（オートロード）
│   │   ├── BackendManager.gd  #   スコア通信・投票API・タイムライン
│   │   ├── StudyDeck.gd       #   デッキ構築・ドロー・バースト判定
│   │   ├── GameSession.gd     #   セッション管理
│   │   ├── AudioManager.gd    #   BGM/SE再生
│   │   └── Enums.gd           #   教科・アイテム列挙型
│   ├── data/
│   │   └── CardData.gd        #   カードデータクラス
│   └── ui/                    # UI/表示層
│       ├── DeskTheme.gd       #   テーマ定数・UIウィジェットファクトリ
│       ├── GameScene.gd       #   メインゲーム画面（カバン構築〜チキンレース〜報告）
│       ├── TitleScene.gd      #   タイトル画面・チュートリアル
│       ├── ProfileScene.gd    #   プロフィール入力
│       ├── ResultScene.gd     #   最終通知表
│       └── SceneTransition.gd #   フェード遷移（オートロード）
└── assets/                    # アセット
    ├── hgrsmp.ttf             #   メインフォント
    ├── bgm_main.wav / se_*.wav #  BGM・SE
    ├── *.png                  #   UI画像群
    ├── split/                 #   アイテム・教科アイコン
    └── tutorial/              #   チュートリアルスライド
```
