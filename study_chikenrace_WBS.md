# Study Chicken Race - WBS (Work Breakdown Structure)

本ドキュメントは、PMBOKのスコープ・マネジメントに基づき、『テスト勉強チキンレース』の開発に必要なすべての成果物と作業を階層的に分解したWBS（作業分解構成図）です。
「時間・予算に追われない環境下で、極限までクオリティを高める」というプロジェクト制約に合わせ、各要素の品質検証アクティビティをWBSに組み込んでいます。

---

## 1. WBS 階層構造

### 1.0 テスト勉強チキンレース (製品全体)
*   **1.1 コア・ゲームロジック (コアエンジン)**
    *   *1.1.1 状態管理モジュール*: 3日間、3時限、各フェーズの状態遷移制御。
    *   *1.1.2 山札・ドロー制御*: シャッフル、ドロー、捨て札再循環、重複（バースト）判定。
    *   *1.1.3 得点・コンボ計算*: 教科ボーナス、連続同教科コンボ加算、アイテム即時効果。
    *   *1.1.4 AI意思決定シミュレータ*: プレイヤー毎の自習戦略（慎重・ギャンブラー等）、チキスタブラフ点申告、ダウト投票確率計算。
    *   *1.1.5 露見確率計算*: いいね投票数と盛り量に応じた数学的カーブ（GDD準拠）の計算、Showdown時の暴露判定。
*   **1.2 ユーザーインターフェース (UI/UX)**
    *   *1.2.1 勉強机テーマ (DeskTheme)*: 共通フォント適用、木製机背景、リングノート、付箋、スマホ枠の描画ユーティリティ。
    *   *1.2.2 メインメニュー*: タイトル、個人成績表示、各メニュー遷移ボタン。
    *   *1.2.3 アカウント設定 (Profile)*: スマホ画面を模したユーザー名登録。
    *   *1.2.4 カバン整理 (Loadout)*: スロット1〜10へのアイテム割当、詳細解説、保存。
    *   *1.2.5 参考書図鑑 (Zukan)*: アンロック済みアイテムの解説、フレーバーテキスト、未解放ロック表示。
    *   *1.2.6 購買部ガチャ (Gacha)*: コイン消費、カプセルマシーン、カプセル開封演出。
    *   *1.2.7 自習室・本編 (Game)*:
        *   1.2.7.1 準備フェーズ: カバン選択肢（3択）カード表示、吸い込み演出。
        *   1.2.7.2 ドローフェーズ: 手札ファン（扇形）配置、ドローめくり、バースト確率表示、睡魔警告。
        *   1.2.7.3 報告フェーズ: 盛り点数調整スライダー、送信ボタン。
        *   1.2.7.4 投票フェーズ: チキスタタイムライン、ライバルつぶやきログ表示、いいね（ダウト）ボタン。
        *   1.2.7.5 答え合わせ (Showdown): 黒板風結果表、はなまる・王冠スタンプ押印、コイン清算。
*   **1.3 ビジュアル ＆ オーディオアセット**
    *   *1.3.1 画像素材*: 机背景、ノート、カード裏表、カレンダー、カバン、スタンプ、黒板などのアンチエイリアス・解像度調整。
    *   *1.3.2 音響素材*: BGM（メイン・結果）、SE（ドロー、クリック、バースト、コンボ、スタンプ押印）。
*   **1.4 品質保証 (QA) ＆ テストプロセス**
    *   *1.4.1 コアテスト*: ロジック検証用単体アサーションテストの実行と自動化。
    *   *1.4.2 クオリティチェック*: ジュース感（手触り・演出テンポ）の評価レビュー。
    *   *1.4.3 統合テスト*: タイトルからゲーム終了、ガチャ、デッキ保存までの通しテスト。

## 2. 成果物定義とクオリティ基準

本プロジェクトでは、時間が無限に使える代わりに「お金をかけずに最高の手触りを作る」ため、WBSの末端成果物（ワークパッケージ）ごとに、厳格なクオリティ基準を定義します。

| WBSコード | 成果物（ワークパッケージ） | クオリティ・手触り基準 |
| :--- | :--- | :--- |
| **1.1.5** | 露見確率計算 | 理不尽な嘘バレを防ぎ、大盛りほど一発逆転されやすいダウト判定カーブの調整テストが完了していること。 |
| **1.2.1** | 勉強机テーマ | 手書き風フォントのアンチエイリアスが効いており、すべてのテキストが滲まずに読めること。紙の影（シャドウ）が柔らかく表現されていること。 |
| **1.2.2** | メインメニュー | ボタンをホバーした際、軽快な「タメ」を伴った拡大と、ランダムな手書き風の傾き（回転）がTweenで表現されていること。 |
| **1.2.7.2** | ドローフェーズ | 山札からカードを引く際、最初は速く、手札に収まる直前に「スッ」と減速するイージングが効いており、カード同士が等間隔で扇状に並ぶこと。 |
| **1.2.7.2** | 睡魔警告 | バースト確率が80%を超えた際、画面周辺が単に赤いだけでなく、プレイヤーの緊張感を煽るようにゆっくりと心拍リズム（約120bpm）でビネットが脈動すること。 |
| **1.2.7.5** | 答え合わせ (Showdown) | 結果が一行ずつ開示される際、はなまるスタンプが「ドスン」と物理的な重み（少しの画面揺れとホコリを模したスケールTween）を持って押されること。 |
| **1.2.6** | 購買部ガチャ | レバーを回した時に一瞬カプセルマシンがガタガタと揺れ、中から飛び出したカプセルをタップすると、殻が二つに割れてカードが手前に飛び出してくるダイナミックな奥行き感がTweenで表現されていること。 |
