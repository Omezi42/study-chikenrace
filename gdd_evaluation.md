## 1. コアコンセプト評価 ★★★★☆

### 🟢 強み

- **「テスト前の牽制し合い」という題材が秀逸**。日本人なら誰もが経験した「全然勉強してないわーｗ」の空気感をゲーム化するという着眼点は、ターゲット層への共感度が極めて高い。
- **チキンレース × ブラフ × デッキ構築**の三層構造により、「リスク管理」「心理戦」「戦略」が一つのゲームループに美しく収まっている。
- **非同期SNS風UI（チキスタ）**がゲーム世界への没入感を大幅に高めている。タイムラインでライバルの報告を見て「👍ダウト」を押すインタラクションは直感的で面白い。

### 🟡 課題

- **「勉強」要素が皆無**。タイトルは「テスト勉強」だが、実際のゲーム内容はカードゲームであり、学習要素は一切ない。ターゲット層（学生時代を懐かしむ大人）には問題ないが、教育ゲームと誤解される恐れがある。
- **「チキンレース」の語感と実態のギャップ**。一般的なチキンレース（度胸試し）からの連想としてはやや複雑で、初見ユーザーにルールの壁がある。

### 🔴 バランス上の重大な懸念

#### (A) デッキのインフレ問題

> [!WARNING]
> **デッキが日を追うごとに肥大化し、バースト確率が下がりすぎる**

GDDの仕様：
- 初期デッキ = 55枚（1が1枚〜10が10枚）
- 毎日アイテムを追加 → 7日目には最低61枚以上

実装コード（[StudyDeck.gd](file:///c:/Users/omezi/Documents/study-chikenrace/scripts/core/StudyDeck.gd#L16-L32)）を確認すると、`reset_for_next_day()` で全カードを山札に戻してシャッフルしている。デッキが膨らむにつれ**同数字の被り確率が希薄化し、後半ほど安全にカードを引ける＝緊張感が薄れる**。

**改善案**:
- 日ごとにデッキサイズ上限を設ける（例：毎日45枚まで圧縮）
- バースト条件を「同数字2枚」→「同数字2枚 OR 場の合計が閾値超過」に拡張
- 日が進むとバーストゾーンが広がる（例：Day5以降は±1の数字もバースト対象）

#### (B) 「時間割」6時間制がテンポを壊す

GDDには「1プレイ3分程度」と書かれているが、実装では [GameSession.gd](file:///c:/Users/omezi/Documents/study-chikenrace/scripts/core/GameSession.gd#L12) の `current_hour` が1〜6まであり、**1日あたりアイテム選択→チキンレース→報告を最大6回繰り返す設計**。これが7日間 = 最大42ラウンドとなり、**「1プレイ3分」は現実的に不可能**。

**改善案**:
- 時間割を3時間に短縮（3×7日 = 21ラウンド）
- または時間割を廃止し、1日1回のチキンレース+複数回のアイテム選択に統合

#### (C) ダウト得点の隠匿が「運ゲー感」を増幅

ダウトの成功/失敗が最終日まで隠されるのはサプライズ演出として優秀だが、**プレイヤーの意思決定へのフィードバックが7日間ゼロ**になり、中間日のモチベーションが低下する。

**改善案**:
- ダウト成功時に「怪しい反応あり…？」等のヒントを翌日表示
- 隠匿得点の一部（概算値）を3日目と5日目に中間発表

#### (D) CPU対戦の「AIパターン」が3体固定で読まれやすい

[AIManager.gd](file:///c:/Users/omezi/Documents/study-chikenrace/scripts/core/AIManager.gd#L6-L10) でライバルは `慎重な優等生`・`ギャンブラー`・`ブラフの達人` の3体固定。プロフィール画面でブラフ傾向まで開示しているため、**2周目以降は最適解が透けて作業プレイ化する**。

**改善案**:
- ライバルプールを8〜10人に拡大し、毎回ランダムに3人選出
- パラメータ（閾値・ブラフ率）に±15%のランダム揺らぎを追加
- プレイヤーの過去成績に応じてAIの攻撃性をスケーリング

## 4. UX / ジュース感 評価 ★★★★★

> [!TIP]
> **このプロジェクト最大の強みは、UIの「おもちゃ感」と「手触り」の良さ**

### 卓越している点

- **物理オブジェクト感**: スマホを「つまんで拡大」、ノートの「ページめくり」、カレンダーの「日めくり」など、すべてのUIインタラクションに物理的な重量感がある。
- **睡魔ゲージの脈動演出**: バースト確率に応じて画面全体が心臓の鼓動のように脈動する演出は、プレイヤーの身体的な緊張を誘発する素晴らしい設計。
- **統一されたビジュアル言語**: [DeskTheme.gd](file:///c:/Users/omezi/Documents/study-chikenrace/scripts/ui/DeskTheme.gd) による一貫したカラーパレット・フォント・コンポーネント設計がプロフェッショナルな印象を与える。
- **コンテクスト遷移の自然さ**: 黒板→ノート→スマホ→カレンダーと、学校生活のメタファーに沿った画面遷移が直感的。

### 改善可能な点

- **チュートリアルのインタラクティブ性**: 現在はテキストベースの説明のみ。「実際にカードを1枚引いてみよう」等のハンズオン型チュートリアルが望ましい。
- **カード引き時のSE・触覚フィードバック**: モバイル向けのバイブレーション対応があるとさらに良い。

## 5. リプレイ性・やり込み評価 ★★☆☆☆

> [!CAUTION]
> **現状、2周目以降のモチベーション維持が最大の課題**

### 問題点

1. **毎回同じ3人のCPU** → 行動パターンが読まれ、最適プレイが固定化
2. **デッキが7日間で徐々に強化される「成長感」はあるが、周回をまたいだ進捗がゼロ** → ローグライクの「1回死んでも何か残る」感覚がない
3. **称号システムは面白いが、コレクション要素やアンロック報酬がない**
4. **デイリーモード未実装**のため、リテンション（定着率）の柱がない

### 改善提案

```
【優先度A】メタ進行要素の追加
- 周回ごとに「通算実績ポイント」を蓄積 → 新しい文房具アイテム・カードスキンをアンロック
- 称号コレクション図鑑（全20種程度）を実装

【優先度B】CPUの多様化
- ライバルプールの拡大 + ランダム選出
- 「○○高校」「△△学園」等の学校名を冠した対戦テーマ

【優先度C】デイリーモードの実装
- GDDに記載済みの「Wordle方式」を優先実装
- 全プレイヤー共通の固定デッキで公平性を担保
- 日替わりランキング → SNSシェア動線
```
